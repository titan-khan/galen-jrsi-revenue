Galen v0 Sequence Diagrams

System interaction flows across six core user journeys, derived from codebase analysis.

1. Hire Specialist — 5-Step Wizard

The core onboarding flow where users create an AI specialist to monitor their business metrics continuously.

sequenceDiagram
    actor User
    participant Wizard
    participant OrgCtx
    participant AI
    participant SpecCtx
    participant DB

    User->>Wizard: Open /specialists/new

    rect rgb(30, 41, 59)
    Note over Wizard: Step 1 Overview
    Wizard->>OrgCtx: getBusinessViewConfigs()
    OrgCtx-->>Wizard: Views list
    User->>Wizard: Select business view
    Wizard->>OrgCtx: getUseCaseCatalog()
    OrgCtx-->>Wizard: Use case templates
    User->>Wizard: Name specialist
    end

    rect rgb(30, 41, 59)
    Note over Wizard: Step 2 Scope
    Wizard->>AI: fetchSuggestedMetrics()
    AI-->>Wizard: Suggested metrics
    User->>Wizard: Select main metric
    User->>Wizard: Add key drivers
    end

    rect rgb(30, 41, 59)
    Note over Wizard: Step 3 Rules
    Wizard->>AI: suggestRules()
    AI-->>Wizard: Thresholds + severity
    User->>Wizard: Configure alert rule
    User->>Wizard: Set scope filter
    end

    rect rgb(30, 41, 59)
    Note over Wizard: Step 4 Knowledge
    User->>Wizard: Upload SLA docs
    User->>Wizard: Add instructions
    end

    rect rgb(30, 41, 59)
    Note over Wizard: Step 5 Alerts
    User->>Wizard: Set channels
    User->>Wizard: Set frequency
    end

    User->>Wizard: Click Hire Specialist
    Wizard->>SpecCtx: createSpecialist(config)
    SpecCtx->>DB: INSERT agents
    DB-->>SpecCtx: specialist_id
    SpecCtx-->>Wizard: Created

    alt Run immediately
        Wizard->>SpecCtx: runSpecialist(id)
        SpecCtx->>DB: INSERT agent_runs
    end

    Wizard-->>User: Show detail page

Key Decision

Business view selection in Step 1 drives which metrics appear in Step 2 and which rules are offered in Step 3.

AI Suggestions

The system auto-suggests metrics, drivers, and monitoring rules based on the selected business view.

v0 Limitation

Specialist creation UI is complete, but persistence to the agents table is not fully wired yet.

2. Specialist Execution Flow

How an AI specialist runs an analysis cycle — from data query through insight generation and streaming results.

sequenceDiagram
    actor User
    participant UI as Detail Page
    participant Svc as RunService
    participant DB
    participant Edge as Edge Fn
    participant Claude

    User->>UI: Click Run Now
    UI->>Svc: runSpecialist(id)

    rect rgb(30, 41, 59)
    Note over Svc: Phase 1 Prepare
    Svc->>DB: Fetch config + rules
    DB-->>Svc: Scope, rules, knowledge
    Svc->>DB: Fetch skill definition
    DB-->>Svc: Frontmatter + body
    Svc->>DB: INSERT skill_executions
    end

    rect rgb(30, 41, 59)
    Note over Svc: Phase 2 Query Data
    Svc->>DB: Query fact_trip
    DB-->>Svc: OTP and delay data
    Svc->>DB: Query dim tables
    DB-->>Svc: Routes, drivers
    Svc->>Svc: Pre-compute stats
    end

    rect rgb(30, 41, 59)
    Note over Edge: Phase 3 AI Analysis
    Svc->>Edge: POST /execute-skill
    Edge->>Edge: Build system prompt
    Edge->>Claude: Stream request

    loop SSE Streaming
        Claude-->>Edge: Delta chunk
        Edge-->>UI: Server-Sent Event
    end
    end

    rect rgb(30, 41, 59)
    Note over Svc: Phase 4 Persist
    Claude-->>Edge: Final response
    Edge->>DB: Store insights
    Edge->>DB: Store recommendations
    Edge-->>Svc: Complete
    end

    Svc->>DB: UPDATE agent_runs
    Svc-->>UI: Done
    UI-->>User: Summary + actions

Progressive Disclosure

Skills use 3 levels: frontmatter (always cached), body (lazy-loaded), resources (on demand).

SSE Streaming

Responses stream in real-time with progress tracking: phase, chunks received, estimated completion.

Skill Pipelines

Architecture supports chaining skills in sequence but pipeline execution is not yet wired in v0.

3. AI Assistant Conversation Flow

Natural language analysis with intent detection, automatic data retrieval, and streaming responses.

sequenceDiagram
    actor User
    participant Chat
    participant Intent
    participant Builder as Context Builder
    participant DB
    participant Edge as Edge Fn
    participant Claude

    User->>Chat: Type message with @mention
    Chat->>Chat: Parse @mentions
    User->>Chat: Send question
    Chat->>DB: Save user message

    rect rgb(30, 41, 59)
    Note over Intent: Intent Detection
    Chat->>Intent: detectIntent(msg)
    Intent->>Intent: Score 8 categories
    Intent-->>Chat: Top intents
    end

    rect rgb(30, 41, 59)
    Note over Builder: Context Injection
    Chat->>Builder: buildQueryContext()
    Builder->>DB: Query fact tables
    DB-->>Builder: Trip, revenue data
    Builder->>DB: Query dim tables
    DB-->>Builder: Routes, drivers
    Builder->>Builder: Deduplicate + stats
    Builder-->>Chat: Data context
    end

    rect rgb(30, 41, 59)
    Note over Edge: AI Response
    Chat->>Edge: POST /assistant
    Edge->>Claude: Stream request

    loop Streaming
        Claude-->>Edge: Text delta
        Edge-->>Chat: SSE event
    end

    Claude-->>Edge: Chart spec
    Edge-->>Chat: Render chart
    end

    Chat->>DB: Save assistant message
    Chat-->>User: Response + chart

8 Intent Categories

revenue, nps, operations, fleet, funnel, agents, general, metadata. Each maps to specific database tables.

@Mention System

Users can @mention specialists and metrics. The mention system resolves IDs and injects relevant data.

Rich Responses

Includes inline Vega-Lite charts, expandable thinking steps, source attribution, and action buttons.

Metric Types

Four types: actionable, result, observational, experimental. Each drives different specialist behaviors.

Status System

Metrics show healthy, warning, or critical status based on target thresholds and anomaly detection.

5. Recommendation Approval & ROI Tracking

From AI-generated recommendation through approval, implementation, and measured business impact.

sequenceDiagram
    actor User
    participant Detail as Specialist Detail
    participant RecCtx as Recommendations
    participant SpecCtx as Specialists
    participant DB
    participant ROI as ROI Tracker

    rect rgb(30, 41, 59)
    Note over Detail: Load Recommendations
    Detail->>DB: Fetch recommendations
    DB-->>Detail: Priority, effort, impact
    Detail-->>User: Recommendation cards
    end

    rect rgb(30, 41, 59)
    Note over User: Approval Flow
    User->>Detail: Review details
    User->>Detail: Click Approve
    Detail->>RecCtx: approve(rec_id)
    RecCtx->>DB: UPDATE status approved
    RecCtx->>SpecCtx: updatePerformance()
    SpecCtx->>DB: UPDATE approval rate
    end

    rect rgb(30, 41, 59)
    Note over User: Implementation
    User->>Detail: Mark Implementing
    Detail->>RecCtx: updateStatus()
    RecCtx->>DB: UPDATE implementing

    User->>Detail: Mark Completed
    Detail->>RecCtx: updateStatus()
    RecCtx->>DB: UPDATE completed
    end

    rect rgb(30, 41, 59)
    Note over ROI: ROI Measurement
    ROI->>DB: Query baseline metrics
    DB-->>ROI: Before: OTP 82%
    ROI->>DB: Query current metrics
    DB-->>ROI: After: OTP 90%
    ROI->>ROI: Calculate delta +8%
    ROI->>DB: UPDATE impact + value
    end

    ROI->>SpecCtx: updatePerformance()
    SpecCtx->>DB: UPDATE value delivered
    Detail-->>User: ROI dashboard

    rect rgb(30, 41, 59)
    Note over User: Alt: Reject
    User->>Detail: Click Reject
    Detail->>RecCtx: reject(rec_id)
    RecCtx->>DB: UPDATE rejected
    RecCtx->>SpecCtx: Adjust accuracy
    end

Status Transitions

pending, approved, implementing, completed (or rejected). Each transition updates specialist performance.

Feedback Loop

Approval rates and ROI data feed back into specialist performance scores for evaluation.

v0 State

ROI tracking UI exists, but automated before/after measurement is not connected to live data yet.

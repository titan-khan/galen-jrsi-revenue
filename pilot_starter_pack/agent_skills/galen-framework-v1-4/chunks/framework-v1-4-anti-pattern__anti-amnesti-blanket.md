# framework_v1.4_anti_pattern — anti_amnesti_blanket

**Source:** `framework_v1.4_anti_pattern`  
**Category:** `anti_amnesti_blanket`

---

ANTI-PATTERN KRITIS: JANGAN berikan amnesti blanket ke semua segmen. Memberikan amnesti ke H1, K1, O1 = MORAL HAZARD. WP yang patuh akan tunda pembayaran menunggu amnesti berikutnya. Ini menghancurkan basis pyramid kepatuhan. Amnesti DIFFERENTIATED, BUKAN BLANKET. H1: TIDAK (loyalitas program). K1: TIDAK (denda <90 hari kecil, masih functional). O1: TIDAK (3-12 bulan masih seimbang dengan pelanggaran). M1: PARSIAL 50-75% denda (denda mulai jadi tembok). M2: PENUH denda (denda 2-4× pokok = tidak akan pernah terbayar). S1: PENGURANGAN BBNKB (lever beda — bukan denda). S2: TIDAK (target = data cleanup). SETIAP amnesti wajib diikuti penegakan setelah deadline.

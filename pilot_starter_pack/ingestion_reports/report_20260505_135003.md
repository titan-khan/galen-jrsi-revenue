## KB delivery — skills-folder approach (PRD deviation)

User chose **Anthropic Skills format** + **Supabase Storage bucket** instead of pgvector embeddings.

**Why deviation:** PRD's Phase 5/6 used OpenAI `text-embedding-3-small` for ~118 chunks → `kb.reference_docs.embedding`. User intends to use OpenRouter for LLM completions in edge functions, but OpenRouter doesn't host embedding models, so the original embedding pipeline was abandoned. Skills-folder approach delivers equivalent retrieval value (Galen agent loads relevant skill folder by topic match instead of vector similarity).

**Skills generated:** [pilot_starter_pack/agent_skills/](../agent_skills/)

|---|---|---|
